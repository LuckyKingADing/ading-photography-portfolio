# ADING 极简个人摄影档案 — 重构规格

## 目标

把现有偏品牌发布页的摄影作品集重构为简单、可持续维护的个人摄影档案。网站服务于个人作品沉淀与审美展示，不承担约拍获客、商业转化、博客或内容营销任务。

设计借鉴 Dan Koe 网站的一句话主张、明确章节、规则化内容列表和高对比黑白基调，但不复制其销售卡片、粗重营销标题或商业内容结构。摄影作品是页面唯一的主要色彩来源。

## 信息架构

网站只保留三类路由：

- `/`：首页，同时承担个人序言、精选照片、作品集索引与个人介绍。
- `/work/[slug]/`：作品集详情页，由同一份系列数据静态生成。
- `/404.html`：无效地址页面。

删除独立 About 页面，不新增独立 Work 页面、分类页、筛选器或 CMS。核心导航固定为 `ADING / Work / About`，其中 Work 与 About 均指向首页锚点。

## 首页结构

首页按以下顺序组织：

1. 极简导航。
2. 英文摄影主张 `Places, light, and people worth remembering.`。
3. 中文说明 `关于旅行、人物和生活片段的个人摄影档案。`。
4. `Selected Photographs`：从所有地点和系列中手动挑选单张照片，按指定顺序混合编排。
5. `Archive`：列出地点、作品集名称、年份与照片数量。
6. `About`：首页底部的一小段个人介绍。
7. 极简页脚。

首页精选照片不使用灯箱。每张照片点击后直接进入它所属的完整作品集。

## 作品集详情页

详情页包含系列名称、地点、时间、类型、摘要和说明，随后按确定顺序纵向展示完整图集。图片不再作为灯箱按钮，不加载客户端画廊脚本。页面末尾提供返回作品集索引的普通链接；当以后存在多个系列时，可由数组顺序派生下一个系列入口。

## 视觉系统

### 色彩

- 背景：`#000000`
- 主文字：`#F5F5F2`
- 次要文字：`#9A9A96`
- 分隔线：`#292929`
- 焦点与交互状态：柔白轮廓与下划线

不使用品牌强调色、渐变、阴影、玻璃效果或装饰面板。照片保持自然彩色，不统一降饱和。

### 字体

- Archivo：英文主标题、栏目标题和导航。
- Noto Sans SC：中文作品名、说明与正文。

删除 Antonio 与 IBM Plex Mono。英文负责主张和栏目层级，中文负责作品名称、描述和内容信息。

### 字号

- 首页主张：手机约 40px，桌面最高 64px。
- 章节标题：24–32px。
- 作品集标题：20–28px。
- 中文正文：手机不低于 16px，桌面约 18px。
- 元信息与图片序号：13–14px。

### 布局与动效

- 内容最大宽度约 1200px。
- 手机水平留白 20–24px，桌面 40–48px。
- 精选照片采用不带卡片外观的两列图集；移动端回到单列。
- 分区只通过留白、标题和细分隔线建立层级。
- 不使用滚动出现动画。
- 链接与图片只保留 150–200ms 的透明度或下划线反馈，并尊重 `prefers-reduced-motion`。

## 数据与代码结构

继续使用 Astro 静态输出和 GitHub Pages。`src/data/series.ts` 是唯一内容数据源。

每张照片保留路径、替代文本、宽度和高度，并可设置 `featuredOrder`。首页把所有系列图片展开后筛选该字段并排序，自动得到精选图集；作品集索引直接由系列数组生成，不维护重复列表。

共享组件只保留真正跨页面使用的基础布局、页头与页脚。首页专属图集和索引直接写在首页页面中，避免单次使用的抽象。删除旧的系列行组件、独立 About 页面、双封面数据、占位状态与灯箱 JavaScript。

## 图片与性能

- 继续使用已经去除元数据的 WebP 公布版本。
- 所有图片显式声明宽高，避免布局偏移。
- 首页第一张精选照片和详情页第一张照片优先加载。
- 其他照片使用原生延迟加载与异步解码。
- 不引入客户端框架、全局状态、运行时 API 或第三方交互库。

## 可访问性

- 保留跳到主要内容链接。
- 使用顺序正确的标题层级。
- 所有有意义照片使用已有的描述性替代文本。
- 普通链接承担导航，不依赖悬停操作。
- 键盘焦点使用高对比可见轮廓。
- 手机正文不低于 16px，触控导航高度不低于 44px。

## 验收

- `npm run check` 零错误、零警告。
- `npm run build` 成功生成首页、作品集页与 404 页面，不再生成 About 页面。
- 375px、768px 与 1440px 下无横向溢出，字号和图集顺序正确。
- 键盘焦点、替代文本、链接和颜色对比可用。
- 首页精选照片进入正确作品集，Archive 信息来自同一数据源。
- GitHub Actions 成功，GitHub Pages 首页和作品集页面返回 HTTP 200。
