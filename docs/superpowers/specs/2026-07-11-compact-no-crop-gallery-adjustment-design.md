# ADING 紧凑无裁切图集 — 调整规格

## 目标

保留现有黑色背景、Archivo 与 Noto Sans SC 字体系统、首页内容顺序和静态 Astro 架构，只重新设计首页 `Selected Photographs` 与作品集详情页图集。

当前桌面首页精选区约 3586px 高，单张图片约 533–1143px 高，纵向间距约 90px。调整后应明显缩短页面、增强图片之间的联系，同时保留足够的摄影编辑感，不退化为普通缩略图墙。

## 硬规则：完整构图

- 所有摄影作品按原始宽高比例显示。
- 图片使用 `width: 100%` 与 `height: auto`。
- 不使用固定图片高度或 `object-fit: cover`。
- 不为对齐版式裁掉人物头部、手部、服装或环境信息。
- 已有图片 `width` 与 `height` 属性继续用于预留空间和避免布局偏移。

## 首页精选区

桌面端采用集中式编辑组合：

- 整个组合最大宽度约 960–1000px，并在内容区域内居中。
- 左侧是一张完整 2:3 主图。
- 右侧四张完整 2:3 图片组成 2×2 网格。
- 主图与右侧四图通过 16–20px 间距形成一个连续视觉块。
- 不再使用错位、跨屏大图和 90px 以上纵向间距。
- 每张照片仍是进入所属作品集的普通链接。
- 图下信息保持简短：系列或地点加两位序号。
- Archive 紧随精选组合，精选区底部留白控制在约 72–96px。

移动端采用两列自然比例网格：

- 全部 5 张精选照片都保留。
- 主图不跨两列，避免重新成为整屏大图。
- 间距约 12px，图下文字保持可读但克制。

## 系列详情页

系列介绍与图集仍使用当前页面，不新增交互或客户端脚本。

- 桌面端图集使用三列自然比例网格。
- 手机端使用两列自然比例网格。
- 桌面横向间距约 20–24px，纵向间距约 32–40px。
- 手机间距约 12–16px。
- 删除奇偶图片左右错位、单张 53rem 宽度与 5–10rem 大间隔。
- 图下注释统一为照片序号、地点与日期，完整语义继续由图片替代文本提供。
- 当前 2:3 竖图均占一列；未来横图依据原始宽高自动跨两列，仍不裁切。

## 视觉特色

首页与详情页使用不同节奏：

- 首页用一大四小的完整构图组合制造视觉入口和继续浏览的欲望。
- 详情页用规则三列、连续编号、地点、日期和细分隔线形成现代联系表与个人档案感。

页面不增加圆角、阴影、渐变、灯箱或装饰动画。照片仍是唯一主要色彩来源。

## 代码边界

- 只修改首页精选区标记、详情页图注和相关 CSS。
- 不修改图片文件、系列顺序、数据源、路由或部署配置。
- 不新增 JavaScript、状态管理、组件库或第三方依赖。

## 验收

- 首页 5 张精选照片和系列页 7 张照片全部完整显示，无裁切。
- 桌面首页精选组合不超过约一屏至一屏半，Archive 明显提前出现。
- 375px、768px、1024px 与 1440px 无横向溢出。
- 手机首页与系列页均为两列，并保留全部照片。
- 所有链接、替代文本、焦点状态与延迟加载行为继续有效。
- Astro 检查、生产构建与 GitHub Pages 部署成功。
