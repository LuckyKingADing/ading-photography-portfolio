# ADING Pinterest 式摄影瀑布流 — 设计规格

## 目标

重新设计首页 `Selected Works` 与四个系列详情页的照片排列，使浏览节奏接近 Pinterest 的等宽多列瀑布流：图片保持原始比例，不再被统一行高约束，后续图片自动进入当前最短列。保留现有黑色背景、中文字体、固定导航、Archive、系列数据与页面文案。

## 方案选择

采用一个无第三方依赖的轻量 Masonry 布局器，而不是 CSS multi-column 或编辑式裁切拼贴。

- CSS multi-column 虽然简单，但会把视觉阅读顺序改为整列优先，不适合已经定义好叙事顺序的摄影系列。
- 编辑式拼贴需要跨列、固定框或裁切，会重新引入人像构图被版式破坏的问题。
- 轻量 Masonry 按 DOM 顺序逐张处理，把每张照片放入当前最短列；DOM、键盘和替代文本顺序继续与数据一致。

## 布局

### 首页 Selected Works

- 桌面使用 3 个等宽列，列间距 16–18px。
- 前三张照片各进入一列，并给三个起始列设置轻微不同的顶部偏移，形成克制的开场错位。
- 后两张按顺序进入当前最短列，使 5 张同为竖幅的人像仍能形成自然的瀑布轮廓。
- 720px 以下使用 2 列并取消开场偏移，保证手机首屏密度和滚动可预期。
- 所有照片继续链接到所属系列，保留现有 hover、焦点和 aria-label。

### 系列详情页

- 1200px 内容宽度下使用 4 列，图片宽度约 288px。
- 721–1024px 使用 3 列，720px 以下使用 2 列。
- 列间和纵向间距统一为 12–18px，形成连续的图片观看流。
- 每张图片按原始宽高比显示；不使用固定高度、`object-fit: cover` 或裁切。
- 图片按数据顺序处理，前三或前四张先从左到右进入首行，之后依次进入当前最短列。

## 实现边界

- 首页精选标记改为单层图片列表，移除仅服务于“一大四小”的嵌套网格。
- 首页与系列列表增加统一的 `data-masonry` 标记。
- 在现有 Astro 布局中加入一个小型客户端脚本：读取列数、间距和图片声明尺寸，计算每个项目的宽度与 `translate3d` 位置，并设置容器高度。
- 使用 `ResizeObserver` 在容器宽度变化时重新排版；不监听滚动，不引入动画库或组件依赖。
- JavaScript 执行前保留普通 CSS Grid 回退；脚本就绪后才切换为绝对定位，避免内容不可见。
- 不修改图片文件、系列顺序、Archive、路由、字体或部署配置。

## 视觉系统

- 继续使用纯黑背景、白色主文字、灰色辅助文字、Archivo 与 Noto Sans SC。
- 不增加圆角、卡片背景、阴影、渐变、图注或浮层按钮。
- 照片仍是页面唯一主要色彩来源。
- 唯一的特色动作是首页首列的轻微错位；系列页保持顶部对齐，让叙事顺序清晰。

## 可访问性与性能

- 图片保留现有中文 alt text、显式 `width`/`height`、首张 eager 与其余 lazy loading。
- DOM 顺序不因视觉排版改变。
- 链接焦点状态保持可见，交互不依赖 hover。
- 布局脚本只在初始化和宽度变化时运行，更新使用 `requestAnimationFrame` 批处理。
- 禁用脚本时仍显示为响应式 CSS Grid，不影响内容访问。

## 验收标准

- 首页仍只有 5 张人像精选；系列页照片数量与顺序不变。
- 桌面首页 3 列、系列页 4 列；平板系列页 3 列；手机全部 2 列。
- 横竖幅照片均保持声明比例，无裁切、无拉伸、无重叠。
- 375px、768px、1024px 与 1440px 均无横向溢出。
- 固定导航、Archive 锚点、系列链接与 Next Series 正常。
- `npm run check`、`npm run build`、`npm audit --omit=dev` 和 GitHub Pages 部署全部通过。
